Ethical Hacker's Reconnaissance Guide

Follow this sequence to build a comprehensive profile of a target URL.

Phase 1: Passive Domain Intelligence

Objective: Understand ownership, history, and the broader organizational footprint without alerting the target.

  • OSINT Framework: Start here to see if there are additional specific categories (like social media or archives) you need to pivot to.
  • Whois DomainTools: Identify the registrar, registration dates, and potential contact info.
  • crt.sh: Search Certificate Transparency logs. This is one of the best ways to find hidden subdomains that have SSL certificates.
  • ViewDNS.info: Run a "Reverse IP" check to see what other sites are hosted on the same server.
  • SpyOnWeb: Look for shared Google Analytics or Adsense IDs to find other websites owned by the same entity.

Phase 2: DNS & Infrastructure Mapping

Objective: Map out the technical "map" of the target.

  • DNSDumpster: Create a visual map of the targetâ€™s DNS records and subdomains.
  • DNS Checker: Verify that the DNS records you found are consistent globally.
  • Robtex: Dive deeper into the relationships between the IP, the domain, and its neighbors.
  • MXToolbox: Check the mail server configuration (MX records) and SPF/DMARC records for potential email spoofing vulnerabilities.

Phase 3: Web Technology & Content Analysis

Objective: Identify the "Stack" (OS, CMS, Plugins) and hidden files.

  • Web-Check: A comprehensive "one-stop" tool to see the technology stack, headers, and basic security flaws.
  • URLScan.io: Take a "safe" look at the site's behavior, scripts loaded, and DOM structure.
  • MySitemapGenerator: Use this to crawl the site and find paths or pages that aren't immediately visible in the navigation.
  • Grep.app: Search for the domain or specific strings in public code repositories to find leaked keys or configuration files.

Phase 4: Attack Surface & Vulnerability Scanning

Objective: Search for open ports and known service vulnerabilities.

  • Pentest-Tools: Run a lightweight vulnerability scan to find low-hanging fruit like outdated versions or missing security headers.
  • SSL Labs: Specifically check the strength of the encryption and look for weak ciphers or expired certs.

Phase 5: IP Reputation & Connectivity

Objective: Check if the server is already compromised or on a watch list.

Objective: Identify the people behind the site for potential social engineering or credential reuse.

  • MetaDataViewer: Run files (PDFs/DOCX) through here to find usernames and software paths.
